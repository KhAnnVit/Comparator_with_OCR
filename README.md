# Comparator

Приложение для автоматизации сверки текста на упаковочных макетах, PDF-шаблонах и данных из Excel-таблиц.

Проект помогает ускорить проверку соответствия упаковок требованиям, которые обычно хранятся в таблицах, технических заданиях или других PDF-документах. Основная идея приложения — дать пользователю удобный инструмент для просмотра PDF, выделения нужных областей, распознавания текста, переноса данных в поля сравнения и подсветки расхождений.

---

## Зачем нужен проект

При проверке упаковок часто приходится вручную сравнивать:

- текст на дизайне упаковки;
- текст в PDF-шаблоне;
- данные из Excel-таблицы;
- требования из технического задания;
- разные версии одного и того же макета.

Полностью автоматизировать такую проверку сложно, потому что на упаковках текст может быть расположен под разными углами, иметь нестандартные шрифты, плохое качество изображения, декоративное оформление или быть разбит на несколько зон. Поэтому приложение не заменяет человека полностью, а помогает убрать большую часть рутинной работы.

Пользователь сам выбирает нужные области, распознаёт текст и проверяет расхождения в удобном интерфейсе.

---

## Основные возможности

- просмотр PDF-файлов;
- перемещение, масштабирование и поворот страницы;
- выделение области на изображении PDF;
- распознавание текста через Tesseract OCR;
- поддержка нескольких языков OCR;
- перенос распознанного текста в поля сравнения;
- частичный перенос выделенного текста через контекстное меню;
- просмотр Excel-таблиц;
- выделение одной или нескольких ячеек Excel;
- перенос текста из Excel в поля сравнения;
- сравнение двух текстовых блоков;
- подсветка найденных расхождений;
- логирование работы приложения.

---

## Текущий сценарий работы

1. Пользователь открывает PDF-файл с упаковкой или шаблоном.
2. В разделе просмотра PDF он выбирает нужную область на изображении.
3. Выделенная область передаётся в раздел OCR.
4. Пользователь запускает распознавание текста.
5. Распознанный текст можно полностью или частично перенести в одно из полей сравнения.
6. Во вкладке Excel пользователь открывает таблицу с требованиями.
7. Из таблицы можно перенести одну или несколько ячеек в поле сравнения.
8. В разделе сравнения пользователь нажимает кнопку сравнения.
9. Программа подсвечивает различия между двумя текстами.

## Описание основных папок и файлов

### `main.py`

Точка входа в приложение. Запускает графический интерфейс и создаёт основной объект приложения.

### `config.py`

Файл с настройками проекта. Здесь могут храниться пути к внешним программам, настройки OCR, список языков распознавания и другие параметры приложения.

### `data/`

Папка для рабочих данных приложения.

* `raw/` — исходные файлы;
* `processed/` — обработанные изображения или промежуточные результаты;
* `app.log` — лог-файл работы приложения.

### `src/gui/`

Папка с окнами и визуальными разделами приложения.

* `main_window.py` — главное окно приложения;
* `pdf_window.py` — раздел просмотра PDF;
* `ocr_window.py` — раздел распознавания текста;
* `compare_window.py` — раздел сравнения текстов;
* `excel_window.py` — раздел просмотра Excel-таблиц.

### `src/models/`

Папка с моделями данных. Эти файлы описывают структуры, которые используются внутри приложения.

* `compare_models.py` — модели для сравнения текста;
* `excel_models.py` — модели для работы с Excel;
* `image_crop_models.py` — модели выделенных областей изображения;
* `image_file_models.py` — модели изображений;
* `ocr_models.py` — модели для результата OCR;
* `pdf_models.py` — модели для PDF-файлов и страниц.

### `src/services/`

Папка с бизнес-логикой приложения.

* `compare_service.py` — логика сравнения текстов;
* `excel_service.py` — работа с Excel-файлами;
* `image_crop_service.py` — обработка выделенных областей;
* `image_file_service.py` — работа с изображениями;
* `ocr_service.py` — распознавание текста;
* `pdf_service.py` — работа с PDF-файлами.

### `src/utils/`

Вспомогательные инструменты проекта.

* `logger.py` — настройка логирования.

### `src/app_controller.py`

Центральный контроллер приложения. Связывает графический интерфейс, состояние приложения и сервисы.

### `src/app_state.py`

Общее состояние приложения. Здесь могут храниться текущие файлы, выбранные области, результаты OCR и данные для сравнения.

### `src/excel_module.py`

Модуль для работы с Excel. Может использоваться как промежуточный слой между интерфейсом и сервисами.

### `src/ocr_engine.py`

Модуль, отвечающий за прямую работу с OCR-движком.

```bash
cd Gradient_comparator
```





## Настройка Tesseract OCR

Для распознавания текста используется Tesseract OCR.

На Windows нужно установить Tesseract отдельно, а затем указать путь к исполняемому файлу в `config.py`.

Пример:

```python
TESSERACT_PATH = r"C:\Program Files\Tesseract-OCR\tesseract.exe"
```

Также нужно установить языковые пакеты для нужных языков. Сейчас для проекта важны:

* русский;
* английский;
* испанский;
* французский;
* арабский.

Пример строки языков OCR:

```python
OCR_LANGUAGES = "rus+eng+spa+fra+ara"
```

## Настройка Poppler

Если в проекте используется конвертация PDF-страниц в изображения через Poppler, его тоже нужно установить отдельно.

После установки путь к Poppler указывается в `config.py`.

Пример:

```python
POPPLER_PATH = r"C:\poppler\Library\bin"
```

## Разделы приложения

### 1. Просмотр PDF

В этом разделе пользователь может открыть PDF-файл и работать со страницей как с изображением.

Доступные действия:

* открыть PDF;
* переключаться между страницами;
* перемещать изображение;
* приближать и отдалять изображение;
* поворачивать страницу на 90 градусов;
* выделять область для OCR.

После выделения область можно отправить в раздел распознавания текста.

### 2. Распознавание текста

В этом разделе отображается выбранная область изображения и результат OCR.

Пользователь может:

* запустить распознавание;
* просмотреть распознанный текст;
* отредактировать текст вручную;
* перенести весь текст в поле сравнения 1;
* перенести весь текст в поле сравнения 2;
* выделить часть текста и перенести её через контекстное меню.

### 3. Сравнение текста

В этом разделе находятся два текстовых поля и кнопка сравнения.

Пользователь может вставить текст из OCR, PDF или Excel, а затем сравнить два блока между собой.

После сравнения программа подсвечивает расхождения красным цветом.

### 4. Просмотр Excel

В этом разделе пользователь может открыть Excel-таблицу с требованиями.

Доступные действия:

* открыть таблицу;
* просматривать содержимое ячеек;
* выделять одну или несколько ячеек;
* переносить данные из выбранных ячеек в поле сравнения;
* использовать контекстное меню для выбора поля сравнения.

## Почему невозможна полная автоматизация

Полная автоматическая сверка упаковок с таблицами пока невозможна без участия пользователя, потому что:

* текст на упаковке может быть повёрнут;
* часть текста может быть мелкой или искажённой;
* OCR может ошибаться в похожих символах;
* один и тот же текст может быть разбит на разные блоки;
* порядок текста на упаковке и в таблице может отличаться;
* дизайн упаковки может содержать декоративные элементы;
* некоторые PDF-файлы не содержат выделяемого текстового слоя;
* требования в Excel могут быть оформлены нестандартно.

Поэтому текущий подход является более надёжным: программа автоматизирует рутинные действия, но пользователь контролирует выбор областей и итоговую проверку.

## Ограничения текущей версии

* качество OCR зависит от качества изображения;
* Tesseract может ошибаться в мелком, повёрнутом или декоративном тексте;
* сложные упаковки всё ещё требуют ручного выбора областей;
* сравнение текста не всегда учитывает смысловые перестановки;
* автоматическое сопоставление Excel и изображения пока не реализовано полностью;
* требуется предварительная настройка Tesseract и Poppler на компьютере пользователя.
